# gsx Phase 3 â€” warm `--watch` on the module core (+ reverse-closure regen)

**Status:** approved design (brainstormed 2026-06-27), ready for an implementation plan.

**One-line:** Migrate `gsx generate --watch`'s warm regeneration path from the legacy
`CachedResolver` (disk-`.x.go`-dependent, full-`packages.Load` rebuild on dep changes)
onto the warm `Module` core, and use the Module's import graph to **regenerate a
changed package's reverse-closure importers** â€” closing the staleness gap where an
importer's `.x.go` was left built against an old dependency.

## 1. Goal & non-goals

### Goal
`--watch` drives one warm `*codegen.Module` per module root. On a settled file change to
package D, it `Invalidate`s D, computes D's reverse-reflexive-transitive importer closure
from the Module's graph, and regenerates **every** package in that closure (D + its
transitive importers), writing byte-identical `.x.go` to disk via the existing hash-gated
`restore`. This (a) retires the `CachedResolver` â€” the warm core's last non-LSP holdout â€”
making watch resolution `.x.go`-free, and (b) fixes the latent correctness gap where
`regen(D)` rewrote only `D.x.go`.

### Non-goals
- **Changing generated bytes.** `Module.Generate â‰ˇ batch` is corpus-proven; the reverse-
  closure set changes *which* `.x.go` files are rewritten, never their content. The corpus
  byte-equivalence gate stays green.
- **Migrating one-shot `gsx generate`.** Only the `--watch` path moves; one-shot generate
  keeps `generateCached` (persistent file cache + batch). That migration is a separate
  later phase.
- **Keeping `generateCached`'s file cache at watch *startup*.** Cold startup moves to
  `Module.Generate` (one pass: writes all `.x.go` AND populates the import graph that
  reverse-closure needs). The cross-run file-cache skip is lost *at watch startup only* â€”
  accepted, since watch is long-lived and startup is paid once; warm regens are incremental.
- **Sub-file / sub-component granularity.** Invalidation/regeneration is per-package-dir,
  as today.

## 2. Background â€” the current watch

`gen/watch.go` runs an fsnotify loop: events fill `pending` (a set of `filepath.Dir(ev.Name)`)
and set `depDirty` if a dep file changed; a debounced `fire` then, if `depDirty`,
`sess.rebuild()`s (rebuilds a `CachedResolver` per root via `packages.Load("./...")`), and
for each `pending` dir calls `sess.regen(dir)`.

`gen/watchsession.go`:
- `watchSession{ resolvers map[string]*CachedResolver }` â€” one warm resolver per module root.
- `newWatchSession`: cold generate via `generateCached` (writes all `.x.go`), then
  `rebuild()` builds a `CachedResolver` per root (`newModuleResolver` â†’
  `codegen.NewCachedResolver` with `./...`).
- `regen(dir)`: `s.resolvers[root].Generate(dir, nil)` â†’ `res.Files` (abs `.x.go` â†’ bytes)
  â†’ `writeFiles(dir, res.Files)` (hash-gated `restore`). On a `cached importer miss` it
  `rebuild()`s once and retries. **Only the changed dir is regenerated.**

The `CachedResolver` loads cross-package types from disk `.x.go` (its `packages.Load`
covers `./...`). It is the path the warm `Module` was built to replace.

## 3. Design

### 3.0 Prerequisite: thread minify config through `Module.Generate`
`Module.Generate`'s emit loop currently calls `generateFile(â€¦ nil, nil, true, true)` â€”
hardcoded built-in **default** minify, ignoring the configured level. `generateFile`
minifies whenever `cssMinify`/`jsMinify` is true (`cssMin=nil` â†’ built-in default;
`cssMin=fullmin.CSS` â†’ full; `false` flag â†’ no minify). The LSP never notices (its
`Package` diagnostic-emit discards output), but **watch writes output**, so it must honor
the user's `effectiveCSSMin()`/`effectiveJSMin()` + levels exactly as one-shot `generate`
does â€” else `--watch` with full/off minify produces wrong bytes.

Add minify settings to `codegen.Options` and store them on the Module; `Module.Generate`
passes them to `generateFile` instead of the hardcoded literals:
```go
type Options struct {
    â€¦ // existing
    CSSMin, JSMin       func(string) (string, error) // custom minifiers (nil = built-in)
    CSSMinify, JSMinify bool                          // whether to minify static CSS/JS
}
```
`Package`'s diagnostic-only emit keeps its hardcoded `nil,nil,true,true` (output discarded,
minify irrelevant there). Because `Options.CSSMinify`/`JSMinify` zero-value is `false`,
existing callers that relied on the old always-true behavior must now set them:
- The corpus equivalence test (`TestModuleMatchesBatchOverCorpus`) Opens its Module with
  `CSSMinify: true, JSMinify: true` (nil funcs) to match the batch args it compares against
  (`â€¦, nil, nil, true, true`) â€” keeping the gate byte-equivalent.
- Watch Opens each Module with `CSSMin: cfg-effectiveCSSMin, JSMin: cfg-effectiveJSMin,
  CSSMinify: cfg.cssMinify, JSMinify: cfg.jsMinify`.
This makes the Module a complete generate core (also the foundation for the later one-shot
`generate` migration), not just an analysis core.

### 3.1 New Module API: `Dependents`
```go
// Dependents returns the reverse-reflexive-transitive closure of dir over the import
// graph: dir plus every project gsx package that transitively imports it. The watch
// session uses it to regenerate every package affected by a change to dir. Returns
// just dir when nothing imports it (or dir is unknown to the graph).
func (m *Module) Dependents(dir string) []string {
	m.mu.Lock()
	defer m.mu.Unlock()
	cl := m.reverseClosure([]string{dir}) // existing internal helper (assumes m.mu)
	out := make([]string, 0, len(cl))
	for d := range cl {
		out = append(out, d)
	}
	return out
}
```
The graph (`importedBy`) is populated whenever the Module analyzes a package, so it is
complete after the cold-startup `Module.Generate` pass over all dirs.

### 3.2 `watchSession` on the Module
`watchSession` holds `modules map[string]*codegen.Module` (per root). Each is built with
`codegen.Open(codegen.Options{ModuleRoot, ModulePath, FilterPkgs, Aliases, FieldMatcher,
Classifier, CSSMin, JSMin, CSSMinify, JSMinify})` â€” the LSP-threaded options (see
`gen/lsp.go` `module()`) **plus the minify config** (Â§3.0), all sourced from `watchConfig`
(`cfg.cssMin`/`cfg.jsMin` are already the resolved `effective*Min()` funcs; `cfg.cssMinify`/
`cfg.jsMinify` the level flags).

- **`newWatchSession`:** discover dirs, `groupByModule`. For each root, `Open` a Module.
  Cold generate = for every dir, `m.Generate(dir)` â†’ write its `.x.go` (Â§3.4). This writes
  all output AND populates every Module's import graph. Emit per-dir startup `cycleResult`s.
- **`regen` is replaced by a closure-aware path** (Â§3.3).
- **`depDirty`:** re-`Open` the affected root's Module and re-run the cold generate for that
  root (fresh `packages.Load` via the lazy `externalImporter`, repopulated graph, rewritten
  `.x.go`). The `cached-importer-miss` retry/`isCachedImporterMiss` logic is removed.

### 3.3 Reverse-closure regeneration (the fire handler)
On `fire` (non-`depDirty`), instead of `for dir := range pending { regen(dir) }`:
```
affected := map[string]bool{}           // dirs to regenerate (deduped union of closures)
for d := range pending {
    if onlyGeneratedRemains(d) { continue }
    m := session.moduleForDir(d)         // resolves root â†’ Module; registers new roots
    m.Invalidate(d)                      // disk-edit invalidation: drop d + closure caches
    for _, dep := range m.Dependents(d) {
        affected[dep] = true
    }
}
for dir := range affected {
    em.cycle(session.regenDir(dir))      // m.Generate(dir) + write .x.go (Â§3.4)
}
```
- `Invalidate(d)` drops d + its closure from `pkgTypes`/`pkgResults` (the graph survives),
  so the subsequent `Generate` of each affected dir re-reads current disk content.
- `Dependents(d)` is read AFTER `Invalidate` (Invalidate does not touch the graph), giving
  the set built against the pre-change graph â€” correct, since a change to d cannot add new
  *importers* of d (only d's own out-edges, refreshed when d re-analyzes).
- The union dedup means a multi-dir change regenerates each affected package exactly once.

### 3.4 Output + diagnostics mapping
`Module.Generate(dir)` returns `(map[string][]byte, []diag.Diagnostic, error)` keyed by
**absolute `.gsx` path**. Convert to the `.x.go` files `restore`/`writeFiles` expects:
```
files := map[string][]byte{}
for gsxPath, b := range out {
    files[strings.TrimSuffix(gsxPath, ".gsx")+".x.go"] = b
}
written, werr := writeFiles(dir, files)   // existing hash-gated restore
```
`cycleResult{Dir, Written, Diags: genDiags, OK: err==nil && !anyErrorDiag(genDiags) && werr==nil, Err}`
mirrors the current shape (diagnostics flow via `Diags`, operational errors via `Err`).

## 4. Data flow (the headline fix)

`components/card.gsx` (component `Card`, used by `pages/page.gsx` via `<components.Card/>`)
is saved:
- fsnotify â†’ `pending = {components}`.
- fire: `m.Invalidate(components)` drops `{components, pages}` from caches (pages imports
  components); `m.Dependents(components) = {components, pages}`.
- `affected = {components, pages}` â†’ `m.Generate(components)` rewrites `card.x.go`,
  `m.Generate(pages)` rewrites `page.x.go` against the new `Card`. **Both** are now current;
  today only `card.x.go` would have been.

## 6. Invariants
- **Output-neutral bytes:** every regenerated `.x.go` is byte-identical to a cold
  `Module.Generate` (and thus to batch / one-shot generate); only the *set* of rewritten
  files grows (closure). Corpus gate green.
- **No stale importer:** after a change to D, every package whose generated code depends on
  D (its reverse-closure) is regenerated in the same cycle.
- **`.x.go`-independent resolution:** watch regen resolves cross-package types from
  skeletons via the Module, never from disk `.x.go` (even though it still *writes* `.x.go`).
- **Disk-edit invalidation:** a saved-file change is reflected because `regen` calls
  `Invalidate(dir)` before `Generate` (no editor `SetOverride`/override in watch).
- **Graph completeness:** the cold-startup `Module.Generate` over all dirs populates every
  importedBy edge, so `Dependents` is complete from the first warm cycle.
- **Concurrency:** the watch loop is single-goroutine (debounced fire); Module calls are
  serialized by its own `analysisMu`. No new shared-state concurrency.

## 7. Testing (per [[gsx-syntax-change-test-coverage]])

- **Minify threading (`internal/codegen`):** `Module.Generate` with `CSSMinify:true` (nil
  func) byte-matches the prior hardcoded behavior; with `CSSMinify:false` the static CSS is
  NOT minified; with `CSSMin: fullmin.CSS` it applies full minify. (A focused codegen test
  on a `<style>`-bearing component; the corpus equivalence gate continues to pass with the
  test Opened at `CSSMinify:true,JSMinify:true`.)
- **Equivalence:** cold-start a watch session over a fixture module + force a regen; assert
  each written `.x.go` is byte-identical to `gsx generate` (one-shot, `generateCached`) on
  the same sources â€” run with a non-default minify level too, to prove the threading. (Reuse
  the existing watch test harness in `gen/`.)
- **Reverse-closure regen (headline):** a 2-package fixture, `pages` imports `components`;
  start watch; modify `components`' component in a way that changes the importer's generated
  call (e.g. rename/add a prop the page passes, or change text the page renders via the
  component); assert **both** `components/*.x.go` and `pages/*.x.go` are rewritten and
  `pages`' new bytes reflect the change â€” and that *before* this phase only the components
  file would have changed (document the contrast in the test).
- **No-op / unrelated:** changing an unrelated leaf regenerates only itself (its
  `Dependents` is just itself); a no-content-change event regenerates nothing new.
- **Dep-change (`depDirty`):** a companion `.go` / go.mod change re-Opens + regenerates
  correctly (output still equivalent).
- **Multi-module:** a session spanning two module roots regenerates each dir against its own
  Module (cross-module refs do not bleed).
- **Existing watch integration tests** (`gen/watch*_test.go`) stay green (adapt only where
  they asserted the now-removed `CachedResolver`/`isCachedImporterMiss` internals).
- **Corpus gate** green; full `gen`/`internal/codegen` suites green.
