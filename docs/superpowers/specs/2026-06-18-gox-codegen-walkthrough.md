# gox — Codegen Walkthrough (hand-written, to validate the design)

**Date:** 2026-06-18
**Status:** Design validation — hand-written generated code, not produced by a tool

Purpose: pin down the **runtime model** and the **`component → func` lowering** by
hand-writing what the generator would emit. If the generated code is clean and the
`Render`/`?` story holds, the design is sound.

## Decision this validates

- **Lazy / streaming runtime, `templ.Component`-compatible.** A `component` lowers
  to `func X(Props) gox.Node`, where `gox.Node` has the **identical method set to
  `templ.Component`** — `Render(ctx context.Context, w io.Writer) error`. So gox
  output is accepted anywhere a `templ.Component` is expected (structpages and
  other templ-ecosystem tools) **without importing templ** (structural interface
  satisfaction).
- The returned `gox.Node` is a **render closure**: the "function that creates the
  element," per the design discussion. **`?` lowers to `return err` from inside
  `Render`** — errors are never ignored or panicked; they stream out of `Render`.
- gox's **built-in behaviors are runtime calls in *generated* code only**
  (`gw.Class`, `gw.Text`, `gw.URL`, …). Source stays sugar (`"cls": cond`,
  `href={…}`). The runtime is implicit in source.

## Runtime sketch (the primitives generated code calls)

```go
package gox

import (
	"context"
	"io"
)

// Node is gox's OWN interface (no alias, no templ import). Its method set is
// IDENTICAL to templ.Component, so a gox.Node is usable anywhere a templ.Component
// is expected, via structural interface satisfaction — keeping gox dependency-free.
type Node interface {
	Render(ctx context.Context, w io.Writer) error
}

// Func adapts a render function to a Node (cf. templ.ComponentFunc).
type Func func(ctx context.Context, w io.Writer) error

func (f Func) Render(ctx context.Context, w io.Writer) error { return f(ctx, w) }

// Writer threads write errors so generated code stays readable (no `if _, err :=`
// on every write). Returns the first error from Err().
type Writer struct {
	w   io.Writer
	err error
}

func W(w io.Writer) *Writer { return &Writer{w: w} }
func (gw *Writer) Err() error { return gw.err }

func (gw *Writer) S(s string)                            // trusted static markup
func (gw *Writer) Text(s string)                         // HTML-escaped text
func (gw *Writer) AttrValue(s string)                    // attribute-escaped value
func (gw *Writer) URL(s string)                          // URL-sanitized attr value
func (gw *Writer) BoolAttr(name string, on bool)         // ` name` if on, else nothing
func (gw *Writer) Class(parts ...ClassPart)              // compose + merge, then write
func (gw *Writer) Spread(ctx context.Context, a Attrs)   // render an attr bag
func (gw *Writer) Node(ctx context.Context, n Node)      // render a child (nil-safe)

// Class composition — GENERATED-only; source uses the "classes": cond sugar.
type ClassPart struct {
	classes string
	on      bool
}
func Class(s string) ClassPart            { return ClassPart{s, true} }
func ClassIf(s string, on bool) ClassPart { return ClassPart{s, on} }

type Attrs map[string]any // attribute bag (spread / implicit rest)
func Raw(s string) Node    // trusted, unescaped HTML (opt-out)
```

## 1. `Card` — element, conditional class, children, interpolation

**Source** (`04_components.gox`):
```go
component Card(title string, featured bool) {
	<section class={ "card", "card-featured": featured }>
		<h2>{title}</h2>
		{children}
	</section>
}
```

**Generated** (`04_components.x.go`):
```go
// Code generated by gox from 04_components.gox; DO NOT EDIT.

type CardProps struct {
	Title    string
	Featured bool
	Children gox.Node // implicit: body references `children`
}

func Card(p CardProps) gox.Node {
	return gox.Func(func(ctx context.Context, w io.Writer) error {
		gw := gox.W(w)
		gw.S(`<section class="`)
		gw.Class(gox.Class("card"), gox.ClassIf("card-featured", p.Featured)) // "cls": cond sugar
		gw.S(`"><h2>`)
		gw.Text(p.Title)                  // {title} → escaped
		gw.S(`</h2>`)
		gw.Node(ctx, p.Children)          // {children} → render child (nil-safe)
		gw.S(`</section>`)
		return gw.Err()
	})
}
```

## 2. `(p UsersPage) Content` — method component + `?` error propagation

**Source** (`11_struct_methods.gox`, simplified):
```go
component (p UsersPage) Content() {        // ctx is ambient — not declared
	<div hx-get={ structpages.URLFor(ctx, UsersList{})? }>
		<h1>{p.Title}</h1>
	</div>
}
```

**Generated:**
```go
// `ctx` is ambient (in scope in every component body); the method takes no params,
// so there is no props struct.
func (p UsersPage) Content() gox.Node {
	return gox.Func(func(ctx context.Context, w io.Writer) error {
		// `?` lowers to a temp + error check that RETURNS FROM Render. `ctx` here is
		// the render ctx, bound to the component's `ctx` param.
		v0, err := structpages.URLFor(ctx, UsersList{})
		if err != nil {
			return err
		}
		gw := gox.W(w)
		gw.S(`<div hx-get="`)
		gw.AttrValue(v0)                  // hx-get is app-controlled → attribute-escaped
		gw.S(`"><h1>`)
		gw.Text(p.Title)
		gw.S(`</h1></div>`)
		return gw.Err()
	})
}
```
`?` is trivially valid because `Render` returns `error`. `ctx` is ambient (in scope
in every component body) — `<p.Content/>` needs no `ctx={…}`.

## 3. `ItemList` — `for` loop + a child component that can fail

**Source:**
```go
component ItemList(items []Item) {
	<ul>
		{ for _, it := range items {
			<li>{it.Name} <Badge kind={it.Kind}/></li>
		} }
	</ul>
}
```

**Generated:**
```go
func ItemList(p ItemListProps) gox.Node {
	return gox.Func(func(ctx context.Context, w io.Writer) error {
		gw := gox.W(w)
		gw.S(`<ul>`)
		for _, it := range p.Items {       // control flow → ordinary Go around writes
			gw.S(`<li>`)
			gw.Text(it.Name)
			gw.S(` `)
			gw.Node(ctx, Badge(BadgeProps{Kind: it.Kind})) // child component; its Render error threads through gw
			gw.S(`</li>`)
		}
		gw.S(`</ul>`)
		return gw.Err()
	})
}
```
A child component error surfaces because `gw.Node` calls the child's `Render` and
records any error (returned by `gw.Err()`), or — for `?`-style short-circuit — an
explicit `if err != nil { return err }`.

## 4. `Box` — implicit `{attrs}` spread + boolean + conditional attr

**Source:**
```go
component Box(padded bool, disabled bool) {
	<div
		class={ "box", "p-4": padded }
		disabled={disabled}
		{ if !padded { data-tight } }
		{...attrs}
	>{children}</div>
}
```

**Generated:**
```go
type BoxProps struct {
	Padded   bool
	Disabled bool
	Attrs    gox.Attrs // implicit: body references `attrs`
	Children gox.Node  // implicit: body references `children`
}

func Box(p BoxProps) gox.Node {
	return gox.Func(func(ctx context.Context, w io.Writer) error {
		gw := gox.W(w)
		gw.S(`<div class="`)
		gw.Class(gox.Class("box"), gox.ClassIf("p-4", p.Padded))
		gw.S(`"`)
		gw.BoolAttr("disabled", p.Disabled)      // type-driven boolean attr
		if !p.Padded {                            // in-tag conditional attribute
			gw.S(` data-tight`)
		}
		gw.Spread(ctx, p.Attrs)                   // {...attrs}
		gw.S(`>`)
		gw.Node(ctx, p.Children)
		gw.S(`</div>`)
		return gw.Err()
	})
}
```

## 5. Calling a component, and an inline slot value (a "block → function")

**Source:**
```go
component Dashboard(items []Item) {
	<Panel header={ <h1>{route.Title(items)?}</h1> }>
		<Card title="Recent" featured>{children}</Card>
	</Panel>
}
```

**Generated:**
```go
func Dashboard(p DashboardProps) gox.Node {
	return gox.Func(func(ctx context.Context, w io.Writer) error {
		// An inline markup VALUE (the `header` slot) becomes its own render closure —
		// this is the "<…> block generates a function" point. Its `?` returns from
		// ITS OWN Render.
		header := gox.Func(func(ctx context.Context, w io.Writer) error {
			title, err := route.Title(p.Items)
			if err != nil {
				return err
			}
			gw := gox.W(w)
			gw.S(`<h1>`)
			gw.Text(title)
			gw.S(`</h1>`)
			return gw.Err()
		})

		card := Card(CardProps{Title: "Recent", Featured: true, Children: p.Children})

		// Render the composed Panel; nested errors stream out via gw.Node → Render.
		return Panel(PanelProps{Header: header, Children: card}).Render(ctx, w)
	})
}
```
Inline **children** are written directly to the current `gw`; inline markup passed
as a **value** (a slot, or a `<>…</>` block bound to a name) becomes a `gox.Func`
closure — a renderer that is itself a `gox.Node`. Slots are therefore `gox.Node`
and lazy by nature (rendered when/if the callee renders them) — no separate
"eager vs lazy slot" decision needed; the streaming model makes slots lazy.

## Design implications surfaced

1. **`templ.Component`-shaped `gox.Node`** → drop-in with structpages and the templ
   ecosystem, no templ import. This is the decisive reason for the streaming model.
2. **`?` → `return err` from `Render`.** Errors are structural and stream out;
   never ignored, never panicked. The unconditional `error` from `Render` is what
   makes `?` always valid in a component body.
3. **Streaming = near-zero allocation** (writes straight to `w`), matching templ's
   performance profile — no per-node tree allocation.
4. **Slots are lazy `gox.Node` closures** for free; inline children are direct
   writes. The "block → function" rule applies exactly to markup *values*.
5. **Built-ins are generated runtime calls** (`gw.Class`/`gw.URL`/`gw.Text`/…),
   never in source.
6. The **`gox.W(w)` error-threading writer** keeps generated code readable while
   still checking every write — a deliberate codegen convenience.

## Resolved

- **`gox.Node` is gox's own interface** (no templ import, no alias); structural
  compatibility provides templ-ecosystem interop.
- **`ctx` is ambient** — implicitly in scope in every component body (the render
  `ctx`); never declared, never a prop, never passed at call sites.
