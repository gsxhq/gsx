# Cached CLI No-op Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make a second, unchanged `gsx generate` on a large real application a genuine persistent-cache hit: no `codegen.GenerateDirs`, no semantic `packages.Load`, byte-identical output, and a measured target of at most 1.5 seconds wall time and 250 MB peak RSS on the baseline machine.

**Architecture:** Split the one-shot cache path into explicit prepare, classify, and commit phases. Prepare freezes the source manifest and Go command context and asks `cmd/go` only for the requested GSX package closure plus semantic configuration roots. Classify assigns each directory a typed hit, miss, or uncacheable decision with a stable reason code. Commit revalidates the Go context, restores hits, batches every miss/uncacheable directory through one semantic generation, and stores only successful cacheable outputs. Internal reporting makes every fallback observable under `-v` and structurally testable without changing the public API.

**Tech Stack:** Go 1.26.1; `cmd/go list -deps -json -compiled`; `golang.org/x/tools/go/packages` through the existing `internal/codegen` module loader; `internal/sourceview`; the existing binary cache store; Go tests/benchmarks; `/usr/bin/time -lp` for the real-workload gate.

## Global Constraints

- The approved design is `docs/superpowers/specs/2026-07-19-cache-hit-recovery-design.md`; preserve its scope and failure semantics.
- Runtime package `github.com/gsxhq/gsx` stays standard-library only. All changes belong to tooling (`gen` and `internal/sourceview`).
- Preserve public `gen.Result`, `gen.Generate`, CLI flags, normal output, generated bytes, diagnostics, and atomic writes. Carry reporting through an unexported helper return; do not add a field to exported `Result`.
- A full cache hit must not open `codegen.Module`, call `codegen.GenerateDirs`, or execute semantic `packages.Load`. One metadata-only `go list` is expected.
- Graph selection is exact. Never restore the unconditional `./...` query and never replace graph selection with directory-name or import-string heuristics.
- Cache validity remains exact: selected GSX and Go sources, transitive Go and GSX dependencies, mutable replacements, module provenance, build context, toolchain/compiler/launcher, generator identity, filters, aliases, renderers, classifier, class merger, and minify settings all stay pinned.
- Standard-library packages and immutable versioned module packages are provenance-pinned and therefore remain cacheable even if `cmd/go` reports `CgoFiles`. Main-module and mutable local-replacement cgo packages remain uncacheable.
- Shared preparation failures fail the cache closed: restore no bytes and write no cache entries, then use one uncached semantic generation only where the existing behavior permits fallback. Go-context provenance/validation failures remain hard errors.
- Corrupt entries are misses and are atomically replaced after successful generation. Per-directory key failures do not poison independently cacheable directories. Cache store write failures remain best-effort and appear only in verbose reporting.
- Do not touch the dirty working checkout at `/Users/jackieli/work/one-learning-gsx`. Real-workload validation uses a disposable local clone.
- Pin all verification to the repository's Go version. Use `GOCACHE=/tmp/...` when needed. `make check` is the inner loop; `make ci` is the authoritative gate; also run `make lint`.
- Use `gopls references` while changing signatures and `gopls check -severity=hint` on every changed Go file.
- Each implementation task is TDD: write the failing assertion, run it and observe the intended failure, implement the smallest complete behavior, rerun focused and neighboring suites, then commit.
- After implementation, an independent adversarial reviewer must build throwaway probes and exercise the cache boundaries, not only inspect the diff.

`SelectedLoadRoots` uses an explicit queue and visited set:

```go
func (manifest *Manifest) SelectedLoadRoots(dirs []string) ([]string, error) {
	var queue []string
	for _, dir := range dirs {
		path, ok := manifest.PackagePathForDir(dir)
		if !ok {
			return nil, fmt.Errorf("sourceview: no manifest package for selected directory %s", dir)
		}
		queue = append(queue, path)
	}
	seen := map[string]bool{"": true, "C": true}
	var roots []string
	for len(queue) > 0 {
		path := queue[0]
		queue = queue[1:]
		if seen[path] {
			continue
		}
		seen[path] = true
		roots = append(roots, path)
		if dir, owned := manifest.PackageDir(path); owned {
			queue = append(queue, manifest.ImportsForDir(dir)...)
		}
	}
	sort.Strings(roots)
	return roots, nil
}
```

Hard-error contract:

- source-manifest build failure -> return error;
- unsafe Go-context fingerprint/provenance failure -> return error;
- semantic commit revalidation failure -> append/return error before restore/generation;
- graph/projection failure -> `cacheReady=false`, shared uncacheable reason, uncached batched generation;
- `ErrUncacheableGoContext` -> `cacheReady=false`, validate, uncached batched generation;
- per-directory key error -> only that directory enters `uncacheable`.

Both `misses` and `uncacheable` are generated together in one ordered, deduplicated call. Only `misses` with a valid key can be stored. Hits are restored before generation, after validation.
