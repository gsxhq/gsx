# Syntax & Usage Docs — Full Coverage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the gsx guide into templ-style per-topic "Syntax and usage" pages with subsections, where examples are merged into the prose and every example is runnable (sourced from golden-tested `examples/*.txtar`).

**Architecture:** Extend the existing `internal/examplegen` generator so a `page:` route in a fixture's `doc` metadata emits a runnable **partial** (`gsx` source + rendered `html` + Playground link) under `docs/guide/syntax/_generated/`. Hand-authored topic pages under `docs/guide/syntax/` pull those partials inline via VitePress `<!--@include-->`. The drift gate (`make ci-examples`) keeps partials honest; the sibling `gsxhq.github.io` repo gets a sidebar group + a `srcExclude` glob.

**Tech Stack:** Go (generator + corpus harness), txtar fixtures, VitePress/markdown (sibling repo).

## Global Constraints

- Runtime root package (`gsx`) stays **standard-library only**. (This plan adds no runtime code; the generator lives in `internal/`, which may use anything.)
- **Don't hand-edit generated files** — `examples.md`, `docs/examples.json`, `playground/server/examples.json`, and the new `docs/guide/syntax/_generated/**` are generated by `cmd/gsx-examples`. Change the `.txtar` source and regenerate.
- **Every example fixture** carries `input.gsx` (package `views`), a `-- doc --` block, an `-- invoke --`, and a `-- render.golden --` (generated, never hand-written). Routed fixtures (those with `page:`) MUST have a `render.golden`.
- Regenerate example goldens with `go test ./internal/corpus -run TestExamples -update`; regenerate the docs artifacts with `make examples`; verify both with `make check` (drops `-count=1`) before claiming done.
- Pin Go to `GO_VERSION` in `.github/workflows/ci.yml` (currently 1.26.1) — a different minor reintroduces gofmt drift.
- Fixture `input.gsx` bodies are **adapted from existing, already-tested corpus cases** named in each task. Adapt for readability (clear identifiers, a realistic invoke), but keep the demonstrated syntax identical to the source case.
- Partials carry **no heading**; the including page owns all headings and prose.

## Authoring Recipe (applies to every content task, Tasks 3–16)

Each content task instantiates these steps with its own page + fixture manifest. This recipe is the body of those tasks — follow it with the task's concrete parameters.

**R1. Create / edit the fixtures.** For each fixture row in the task manifest:
- Create `examples/<file>.txtar` (or edit the named existing file). Copy `input.gsx` and `invoke` from the named **source corpus case** (`internal/corpus/testdata/cases/<case>.txtar`), set `package views`, and adapt identifiers for clarity.
- Prepend the `-- doc --` block using this exact shape:
  ```
  -- doc --
  name: <Name>
  summary: <one-line summary>
  category: <Category>
  page: <page-slug>
  pageOrder: <NN>
  ```
  `category` keeps the example in the playground dropdown; `page`/`pageOrder` route it to the syntax page. Existing reused fixtures: **add** `page:`/`pageOrder:` lines to their `doc` block (leave `category`/`order` as-is).

**R2. Generate render goldens.** Run `go test ./internal/corpus -run TestExamples -update`. This compiles + renders each fixture through the real pipeline and writes `render.golden`. An example that produces diagnostics fails — fixtures must be clean.

**R3. Regenerate docs artifacts.** Run `make examples`. This writes `examples.md`, `docs/examples.json`, `playground/server/examples.json`, and the routed partials under `docs/guide/syntax/_generated/<page>/`.

**R4. Write the page.** Create/edit `docs/guide/syntax/<page>.md` — hand-authored prose with the task's subsection headings, each followed by the matching include:
  ```md
  ## <Subsection>

  <prose explaining the feature>

  <!--@include: ./_generated/<page>/<NNN>-<slug>.md-->
  ```
  (`<NNN>` is the zero-padded `pageOrder`, `<slug>` is the lowercased hyphenated `name` — check the generated filename under `_generated/<page>/`.)

**R5. Verify.** Run:
- `go test ./internal/corpus -run TestExamples` (no `-update`) → PASS.
- `make ci-examples` → no drift (exit 0).
- If the sibling site is checked out: `cd ../../gsxhq.github.io && GSX_DOCS_SRC=<this-worktree> npm run docs:build` (or `npm run build`) → builds clean. If not available, skip and note it.

**R6. Commit.** `git add examples/ docs/ && git commit -m "docs(syntax): <page> page"`.

// slug lowercases name and collapses non-alphanumerics to single dashes.
func slug(name string) string {
	var b strings.Builder
	dash := false
	for _, r := range strings.ToLower(name) {
		if (r >= 'a' && r <= 'z') || (r >= '0' && r <= '9') {
			b.WriteRune(r)
			dash = false
		} else if !dash {
			b.WriteByte('-')
			dash = true
		}
	}
	return strings.Trim(b.String(), "-")
}

// partialRelPath is "<page>/<NNN>-<slug>.md".
func partialRelPath(e Example) string {
	n := e.PageOrder
	if n == 0 {
		n = e.Order
	}
	return filepath.Join(e.Page, fmt.Sprintf("%03d-%s.md", n, slug(e.Name)))
}

// RenderPartial emits one routed example as an include partial: source gsx,
// rendered html, and a Playground link. No heading — the page owns headings.
func RenderPartial(e Example) []byte {
	var b strings.Builder
	b.WriteString("<!-- GENERATED from examples/*.txtar by cmd/gsx-examples — do not edit. -->\n\n")
	for _, f := range e.Files {
		if len(e.Files) > 1 {
			b.WriteString("**" + f.Name + "**\n\n")
		}
		b.WriteString("```gsx\n")
		body := f.Body
		if !strings.HasSuffix(body, "\n") {
			body += "\n"
		}
		b.WriteString(body)
		b.WriteString("```\n\n")
	}
	b.WriteString("Renders:\n\n```html\n")
	r := e.Render
	if !strings.HasSuffix(r, "\n") {
		r += "\n"
	}
	b.WriteString(r)
	b.WriteString("```\n\n")
	b.WriteString("[▶ Open in Playground](/playground#try=" + tryPayload(e.Source, e.Invoke) + ")\n")
	return []byte(b.String())
}
```

// Generate loads examplesDir, writes routed examples as partials under
// partialsDir, the unrouted gallery to mdPath, and presets to each jsonPaths.
func Generate(examplesDir, mdPath, partialsDir string, jsonPaths ...string) error {
	exs, err := Load(examplesDir)
	if err != nil {
		return err
	}
	// Rebuild the partials tree from scratch so renamed/removed examples leave
	// no orphan partials (the drift check would otherwise miss the deletion).
	if err := os.RemoveAll(partialsDir); err != nil {
		return err
	}
	var gallery []Example
	for _, e := range exs {
		if e.Page == "" {
			gallery = append(gallery, e)
			continue
		}
		full := filepath.Join(partialsDir, partialRelPath(e))
		if err := os.MkdirAll(filepath.Dir(full), 0o755); err != nil {
			return err
		}
		if err := os.WriteFile(full, RenderPartial(e), 0o644); err != nil {
			return err
		}
	}
	if err := os.WriteFile(mdPath, RenderMarkdown(gallery), 0o644); err != nil {
		return err
	}
	pj, err := presetsJSON(exs) // playground shows ALL examples
	if err != nil {
		return err
	}
	for _, p := range jsonPaths {
		if err := os.WriteFile(p, pj, 0o644); err != nil {
			return err
		}
	}
	return nil
}
