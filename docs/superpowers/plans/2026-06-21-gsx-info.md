## Goal

`gsx info` prints the active pipeline filter table — every resolved filter, its owning
package, and any SHADOWING (a later filter package overriding an earlier same-named
filter, per last-wins). This makes the otherwise-implicit `WithFilters` registry
inspectable (the "correct, not surprising" guarantee), and surfaces the shadow that a
future `gsx vet` would warn about.

## Conflict-safety (a parallel agent reorganizes tests)

`gen/info_test.go`). No edits to existing `*_test.go`. Additive codegen seam only
(no error-format changes — those would collide with the corpus `diagnostics.golden`).

## Single task: `ResolveFilters` seam + `gsx info` command

### Part A — codegen seam (`internal/codegen/filters.go`)
A PUBLIC function returning the resolved filters as structured data:
```go
// FilterInfo describes one resolved pipeline filter, for `gsx info`.
type FilterInfo struct {
    Name     string   // template name (first-rune-lowered), e.g. "upper"
    Pkg      string   // winning package import path
    Func     string   // exported Go func name, e.g. "Upper"
    Param    bool     // parameterized func(Args...) func(T) R; else bare func(T) R
    Shadows  []string // import paths of EARLIER same-named filters this one overrides
}

// ResolveFilters harvests the filter packages (in order, last-wins) and returns the
// resolved table sorted by Name, recording which earlier same-named filters each
// winner shadows. dir anchors the go/packages load against the module's go.mod.
func ResolveFilters(dir string, filterPkgs []string) ([]FilterInfo, error)
```
- Reuse the existing harvest machinery (`loadFilterTableMulti` harvests in pkg order
  with last-wins). Factor a shared helper if cleanest: e.g. `harvestFilters(dir,
  pkgPaths) (map[name][]entry-with-pkg, error)` collecting ALL same-named entries in
  package order; `loadFilterTableMulti` keeps using the winner (last); `ResolveFilters`
  builds `FilterInfo` (winner = last entry; `Shadows` = the pkgPaths of the earlier
  entries for that name). Keep `loadFilterTableMulti`'s behavior byte-identical (don't
  regress Task-1 of WithFilters).
- Default empty `filterPkgs` → `[stdImportPath]` (match `GeneratePackageWithFilters`).
- Errors: a bad package path → a clean error (reuse the per-package guards).

### Part B — `gen info` command (`gen/`)
- `runInfo(stdout io.Writer, dir string, filterPkgs []string) int` — resolves filters
  via `codegen.ResolveFilters(dir, filterPkgs)` and prints, e.g.:
  ```
  gsx <version>

  Filter packages (last-wins):
    github.com/gsxhq/gsx/std

  Filters (12):
    default    std.Default      param
    join       std.Join         param
    lower      std.Lower        bare
    truncate   std.Truncate     param
    upper      std.Upper        bare
    ...
  ```
  When a filter shadows another, mark it, e.g.
  `upper      myfilters.Upper  bare   (shadows github.com/gsxhq/gsx/std)`.
  Sorted by name; aligned columns (use text/tabwriter for clean alignment). Use the
  short `pkg.Func` form for the qualified name (last path element + `.` + Func) but
  print the full shadowed import path so it's unambiguous.
- Wire into `runConfig`'s dispatch: add `case "info": return runInfo(stdout, ".",
  cfg.filterPkgs)` (resolve against cwd; honor `-C` which already chdir'd). On a
  resolve error → print to stderr, exit 1.
- Add `info` to `printUsage`'s command list with a one-line description.
- `cmd/gsx` gets it for free (stock = std).
